# Elysia 项目总规划（可直接执行）

## 一、项目摘要
1. 项目目标：打造一个“低门槛记录 + 高价值探索”的情绪与思维网站，核心是让用户愿意开始写、持续写、并从记录中发现自我与共鸣。
2. 核心模块：`Elysia Home`（个人记录）与 `Elysia Universe`（公开探索），并通过 `MindMap` 连接情绪、金句、灵感、评论派生记录。
3. 体验原则：默认极简、渐进展开、一步可发、深度能力按需显现，避免传统记录软件“流程重、门槛高、懒得写”。
4. 技术基线：`Node.js + Fastify + PostgreSQL + Redis + Cloudflare R2 + WebSocket + OpenAPI 3.1`。
5. 部署基线：阿里云单机 `Docker Compose + Caddy + cloudflared(CF Tunnel)`；本地无 OAuth（开发假用户），线上接 LinuxDoConnect OIDC。
6. 视觉基调：水晶花液态玻璃风格，借鉴爱莉希雅“无瑕、纯净、温柔”的气质表达，不使用角色立绘素材。

## 二、产品与交互详细设计

### 2.1 Home（首要入口，反拖延核心）
1. 入口默认只有一个主动作：`写下此刻心情/思路`。
2. 首访使用 3 屏引导，可全程跳过并记住；回访仅保留一句温柔欢迎语。
3. 默认输入为“一句话文本”，只需一句即可发布。
4. 输入首句后字段智能逐步出现（每次 1-2 项）：金句、附加情绪、描述、图片/画布、时间、地点、公开设置。
5. 卡住引导策略：编辑空白停留 8 秒触发 1 条轻提示，同时常驻“给我灵感”手动按钮。
6. 自动保存：输入后 1.5 秒防抖保存草稿。
7. 草稿策略：每用户最多保留 5 份草稿，顶部小提示支持一键恢复上次。
8. 离线能力：支持离线写草稿，恢复网络后自动重试同步。
9. 发布反馈：短反馈动效后立即回到主视图，不强制填写全部字段。
10. 编辑规则：正文发布后 30 天内可改；公开/私密切换无时间限制。

### 2.2 Universe（公开探索，非列表沉浸）
1. 形态为无限二维上下左右探索，不采用普通直列列表。
2. 每个视口固定 `1 主焦点 + 2 副焦点`，保证视觉中心清晰。
3. 焦点更新节奏：视口停留 20 秒后微调，避免频繁跳动。
4. 显示策略：远处模糊占位，停止后逐步清晰。
5. 内容节奏：每日主题星团全天稳定；共鸣脉冲每 2 小时刷新。
6. 排序策略：热度 + 新鲜度混排，并保留中等个性化权重（40%）。
7. 互动方式：hug/heart 拖拽到目标内容触发反馈与实时计数更新。
8. 内容卡片默认精简展示：心情短语 + 金句 + 时间 + 分享者；详情再展开评论与派生关系。
9. 评论他人内容后自动生成“我的新记录”，并与原帖双向关联。

### 2.3 MindMap（清晰视图 + 深度发现）
1. 进入规则：首次进入网站默认 Home；若 30 分钟内有新记录，下次优先进 MindMap。
2. 用户控制：允许一键回 Home，并记忆入口偏好。
3. 模式切换：顶部一键 `简洁模式/深度模式`，并记忆上次模式。
4. 默认视图范围：本人记录 + 公开弱连接。
5. 布局算法：关系力导向 + 时间环叠加。
6. 节点类型：记录、金句、附加情绪、主题簇、评论派生、事件触发。
7. 关系类型：关键词相似、向量语义相似、时间邻近、互动共鸣、人工补链。
8. 清晰策略：高密度自动聚类，缩放/停留后再解聚。
9. 编辑策略：正文 30 天内可改；标签和关联永久可改；所有变更留审计轨迹。
10. 视图能力：关系图 + 时间轴 + 主题树可自由切换（简洁/深度）。

### 2.4 7天激活剧本（每天≤1分钟）
1. D1：一句话发布，建立“我可以马上记录”。
2. D2：补一项（仅金句或附加情绪二选一）。
3. D3：点选 1 条关联，形成第一条思维链。
4. D4：添加 1 张图或 1 段手写。
5. D5：浏览主题星团并完成一次互动。
6. D6：查看周轨迹并确认一个“本周关键词”。
7. D7：里程碑反馈（徽章 + 主题微动效 + 深度入口提示，不锁核心功能）。

## 三、UI 设计系统规范（给 Gemini 的实现基线）
1. 风格关键词：Crystal Bloom、Liquid Glass、Soft Glow、Calm Motion。
2. 颜色体系：冰晶白、晨雾蓝、珊瑚粉、浅金高光；避免高饱和与重紫主导。
3. 材质体系：高斯模糊玻璃层、柔边高光、微色散描边、低噪点纹理背景。
4. 组件体系：气泡节点、液态卡片、渐进字段组、焦点标签、拖拽表情、里程碑卡片。
5. 动效体系：主形变 260-320ms，次反馈 120-180ms，背景呼吸 6-10s 周期。
6. 信息层级：速记层永远在第一层，深度工具默认折叠在第二层之后。
7. 文案体系：温柔、陪伴、非命令式；可使用指定短句风格。
8. 三端适配：手机优先单手可达；iPad 强化手写分栏；PC 强化探索与多视图切换。
9. 可访问性：高对比模式、减弱闪烁开关、触控目标最小尺寸、关键操作可键盘触发。
10. 性能感知：所有大图先占位后清晰，禁止“白屏等待”。

## 四、系统架构与部署方案

### 4.1 服务拆分
1. API 服务：Fastify，负责鉴权、读写、权限、业务编排。
2. Worker 服务：处理图片转码、向量计算、周报汇总、导出任务。
3. Queue：异步任务编排与削峰。
4. DB：PostgreSQL 主存。
5. Cache：Redis（会话、热点、计数、限流、队列辅助）。
6. Object Storage：Cloudflare R2（图片、画布资源、导出文件）。
7. 反向代理：Caddy。
8. 外网入口：Cloudflare Tunnel（cloudflared）+ CF DNS。

### 4.2 环境策略
1. 本地：假用户切换，不启 OIDC，便于前后端联调。
2. 线上：LinuxDoConnect OIDC，完整会话与权限校验。
3. 发布策略：蓝绿或滚动重启；先灰度后全量。
4. 数据策略：每日全量备份 + 每小时增量（保留 3 天）+ 异常快照。

## 五、数据模型与存储策略
1. 用户域：`users`、`identities`、`sessions`、`user_preferences`。
2. 记录域：`records`、`record_quotes`、`record_emotions`、`record_links`、`drafts`。
3. 社交域：`comments`、`reactions`、`follows`（可选）、`notifications`。
4. 图谱域：`mindmap_nodes`、`mindmap_edges`、`insight_snapshots`。
5. 媒体域：`media_assets`、`drawing_docs`、`media_variants`。
6. 地点时间域：`locations`、`record_time_meta`。
7. 治理域：`reports`、`moderation_actions`、`sanction_logs`。
8. 统计域：`access_events`、`endpoint_minute_stats`、`user_frequency_stats`、`alert_events`。
9. 约束：单条最多 4 图，每图 ≤ 5MB，每用户全站最多 60 图。
10. 隐私：地点默认城市级打码，可手动更细；打码文本自然融入，不做生硬标签化。

## 六、API 与实时事件（对前端契约）
1. Auth：`/auth/oidc/login`、`/auth/oidc/callback`、`/auth/logout`、`/auth/dev/switch-user`。
2. Record：`POST /records`、`PATCH /records/:id`、`PATCH /records/:id/visibility`、`GET /records/:id`、`GET /home/feed`。
3. Comment/Derive：`POST /records/:id/comments`（返回派生记录 ID 与关联信息）。
4. Universe：`GET /universe/viewport`、`GET /universe/focus`、`GET /universe/hot`、`GET /universe/recent`。
5. MindMap：`GET /mindmap/me`、`GET /mindmap/:recordId`、`POST /mindmap/manual-link`。
6. Media：`POST /media/upload-sign`、`POST /media/complete`、`POST /drawings`、`GET /media/:id/variants`。
7. Insight：`GET /insights/emotion-trajectory`、`GET /insights/theme-evolution`、`GET /insights/resonance-network`。
8. Nudge：`GET /nudges/recommendations`、`PATCH /nudges/settings`、`POST /nudges/feedback`。
9. Governance：`POST /reports`、`GET /admin/reports`、`POST /admin/reports/:id/resolve`、`POST /admin/users/:id/sanction`。
10. Analytics：`GET /admin/analytics/overview`、`/endpoints`、`/users-frequency`、`/alerts`。
11. WebSocket 事件：`record.created`、`record.updated`、`reaction.updated`、`focus.shifted`、`moderation.changed`。

## 七、AI 与推荐能力边界
1. 明确禁止：AI 自动生成用户“描述/金句”正文。
2. 允许能力：标签建议、周报总结、轨迹洞察、主题聚类建议。
3. 接口模式：用户自填模型地址与密钥由客户端直连，不经服务器代发。
4. 举报机制：支持“疑似 AI 生成”举报，管理员确认后记次。
5. 处罚策略：3 次确认 7 天禁言，5 次封禁，审核 SLA 为 72 小时。
6. 关联算法：关键词规则 + 向量语义混合建链，实时增量 + 每日校准。

## 八、性能、负载与监控优化
1. 目标：Universe 首屏 ≤ 2.5s，核心交互 < 120ms。
2. 并发目标：500 DAU / 峰值 100 并发下稳定运行。
3. 异步化：图像处理、向量计算、导出、周报全走队列 Worker。
4. 缓存：热点内容与焦点节点 Redis 缓存，短 TTL + 主动失效。
5. 查询优化：关键查询加联合索引与分页游标，避免大偏移扫描。
6. 媒体优化：多尺寸 WebP/AVIF，先模糊占位再清晰。
7. 监控维度：全站、接口、用户访问频率三层。
8. 告警策略：阈值告警 + 突增检测。
9. 日志保留：明细 7 天，聚合 6 个月。
10. 看板：系统健康、Universe 热点、用户频率、风控异常、队列积压。
11. 限流：按用户 + IP 双维令牌桶。
12. 人机挑战：仅高频异常时触发 Cloudflare Challenge。

## 九、安全、隐私与治理
1. 认证授权：OIDC + RBAC（普通用户/管理员）。
2. 会话安全：短会话令牌 + 刷新机制 + 敏感操作二次校验（管理端）。
3. 上传安全：签名上传、类型校验、大小限制、恶意文件拦截。
4. 数据安全：敏感字段加密存储，最小权限访问。
5. 审计合规：管理员操作全审计，处罚有依据可回溯。
6. 隐私控制：公开开关默认记忆上次选择，用户可随时切换。
7. 个性化数据：默认开启并可关闭（站内行为 + 阅读停留偏好）。

## 十、测试计划与验收标准
1. 单元测试：长度规则、编辑期限、公开开关、图片配额、草稿限制、派生逻辑。
2. 集成测试：发布-补全-关联-评论派生-互动-举报全链路。
3. 契约测试：OpenAPI 与前端字段一致性、错误码与分页规则。
4. E2E 测试：手机/iPad/PC 三端关键路径。
5. 性能测试：视口加载、焦点刷新、WS 推送、图片切换与拼图详情。
6. 稳定性测试：断网续写、重连同步、队列拥塞恢复、缓存击穿保护。
7. 安全测试：OIDC 流程、权限绕过、限流、生效与告警。
8. 灾备演练：全量/增量恢复、异常快照恢复、Tunnel 中断恢复。

## 十一、里程碑与交付节奏
1. M1（后端基础）：鉴权、记录域、媒体上传、草稿与渐进流程、OpenAPI 初版。
2. M2（核心体验）：Universe 视口与焦点、MindMap 基础、评论派生、实时互动。
3. M3（深度能力）：洞察、7天激活、引导系统、导出、治理后台。
4. M4（稳定上线）：压测、监控告警、备份恢复演练、灰度上线。
5. 每两周一轮 A/B：单轮控制 1-2 个变量，持续优化“愿意写”指标。

## 十二、可继续扩展（后续版本候选）
1. 情绪天气与主题皮肤随记录动态变化。
2. 记忆胶囊（未来日期唤醒历史片段）。
3. 双人接龙记录与同频房间（可选匿名）。
4. 语音速记与轻量转写（可选）。
5. 更细粒度危机关怀资源与区域化热线卡片。
6. 个人“成长档案馆”自动编排（季度/年度）。

## 假设与默认
1. 当前计划覆盖后端与产品规格，前端由 Gemini 按本契约实现视觉与动效。
2. 默认统一高动效；若实测普遍掉帧，再按机型灰度降级。
3. 计划已决策完整，可直接进入实施拆解。
