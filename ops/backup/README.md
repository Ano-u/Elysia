# 备份与快照策略

## 策略目标

- 每日全量备份
- 每小时增量快照（保留 3 天）
- 异常事件触发紧急快照

## 脚本

- `backup.ps1`: 全量备份（PostgreSQL）
- `incremental.ps1`: 小时级逻辑快照（`pg_dump -F c`）
- `emergency-snapshot.ps1`: 异常快照触发标记

## 建议计划任务（Windows）

1. 全量备份：每天 03:15 执行 `backup.ps1`
2. 增量快照：每小时执行 `incremental.ps1`（保留 3 天）
3. 清理策略：保留最近 3 天增量、最近 3 份全量

## 恢复演练建议

1. 每周做一次“全量 + 最新增量”恢复演练。
2. 恢复后跑健康检查并比对核心表数量。
